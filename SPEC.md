# 大文件文本编辑器 - 规格文档

## 1. 项目概述

- **项目名称**: FastEdit - 大文件文本编辑器
- **项目类型**: 桌面应用程序 (Electron)
- **核心功能**: 支持800MB+大文件快速加载、虚拟滚动的文本编辑器
- **目标用户**: 需要处理大型日志文件、代码文件的技术人员

## 2. UI/UX 规格

### 2.1 布局结构

- **单窗口应用**: 包含标题栏、菜单栏、主编辑区域、状态栏
- **标题栏**: 显示当前文件名和应用名称
- **菜单栏**: 文件菜单（打开、保存、另存为、退出）、编辑菜单（撤销、重做、剪切、复制、粘贴）、视图菜单（放大、缩小、重置缩放）
- **主编辑区域**: 左侧行号栏 + 右侧文本编辑区
- **状态栏**: 显示文件大小、光标位置、编码信息

### 2.2 视觉设计

- **主题**: 深色主题（护眼）
- **主色调**:
  - 背景色: #1e1e1e
  - 文本色: #d4d4d4
  - 行号背景: #252526
  - 行号文字: #858585
  - 选中行: #2a2d2e
  - 光标: #aeafad
  - 状态栏背景: #007acc
- **字体**: Consolas, 'Courier New', monospace，16px
- **行高**: 24px
- **间距**: 左侧行号栏宽度 60px

### 2.3 组件

| 组件 | 描述 | 状态 |
|------|------|------|
| 行号栏 | 显示行号，右对齐 | 固定在左侧 |
| 文本区 | 主编辑区域 | 可滚动 |
| 光标 | 细线竖杠，2px宽 | 闪烁动画 |
| 状态栏 | 显示文件信息 | 固定底部 |

## 3. 功能规格

### 3.1 核心功能

1. **大文件加载**
   - 使用内存映射文件（Memory-mapped files）技术
   - 支持加载 800MB+ 的纯文本文件
   - 加载进度显示
   - 只读模式访问原始数据

2. **虚拟滚动**
   - 只渲染可视区域内的行（前后各预留10行缓冲）
   - 平滑滚动
   - 行高固定：24px

3. **文本显示**
   - 按需分页加载可见行
   - 缓存已渲染的行
   - 支持大文件的行号计算

4. **基础编辑功能**
   - 光标定位（点击、方向键）
   - 撤销/重做（支持100步）
   - 文本选择
   - 复制/粘贴

5. **按需渲染**
   - 懒加载可见区域内容
   - 后台预加载临近行
   - 内存缓存机制

### 3.2 用户交互

- **打开文件**: Ctrl+O 或菜单
- **保存文件**: Ctrl+S
- **撤销**: Ctrl+Z
- **重做**: Ctrl+Y
- **光标移动**: 方向键、Home/End、PageUp/PageDown

### 3.3 性能目标

- 800MB文件加载时间: < 5秒
- 滚动帧率: 60fps
- 内存占用: < 500MB（针对800MB文件）

## 4. 技术规格

### 4.1 技术栈

- **框架**: Electron
- **前端**: HTML/CSS/JavaScript
- **文件处理**: Node.js Buffer.slice + 自定义内存映射
- **打包**: electron-builder (portable模式)

### 4.2 内存映射实现

```
- 使用 Node.js fs.open + fs.read + Buffer 实现模拟内存映射
- 将大文件分块读取到内存缓存
- 按需加载指定行范围
- 行索引缓存加速定位
```

### 4.3 虚拟滚动算法

```
- 滚动位置 -> 起始行号计算
- 起始行号 -> Buffer读取范围
- Buffer -> 行解析 -> DOM渲染
- 滚动事件触发重新计算
```

## 5. 便携部署

- 使用 electron-builder 的 portable 模式
- 输出单个 .exe 文件
- 无需安装，直接运行
- 配置文件存储在 exe 同目录
